Auto Machinery Inventory System

Full-stack web platform for managing thousands of automotive products efficiently.

Project Objective

The Auto Machinery Inventory System was developed to support a growing automotive parts supplier in Singapore by digitalizing their inventory process. The objective was to simplify search, categorization, and updates for thousands of products, ensuring smooth operations for both staff and clients.

Vision

Our vision was to build a platform that could scale with product growth, enable seamless filtering and CRUD management, and reduce reliance on manual Excel-based workflows.

My Role in the Project

  • Designed and implemented key navigation components such as Home, About Us, and Contact Us pages, with content dynamically editable via an admin panel.
  • Built a Content Management System like admin interface to manage static content using MySQL, enabling updates without touching code.
  • Developed product management functionality including Create, Read, Update, and Delete (CRUD) features in the admin panel, which reflect in real time on the customer-facing product listings.
  • Engineered a shopping cart system that handles adding, viewing, and removing products linking frontend interactions to backend logic.
  • Ensured the system could handle a large dataset of automotive products efficiently by implementing pagination, search filters, and optimized queries.

Challenges & Key Learnings

Working with thousands of products required efficient backend logic and optimized queries. I learned to use indexed DB fields, lazy loading components, and how to break monolithic UI into reusable modules.

Working with thousands of automotive products required designing an efficient backend with optimized SQL queries and indexed database fields to ensure fast response times. I implemented lazy-loaded components and broke down the monolithic UI into reusable, modular layouts for better scalability and performance.

Additionally, I built a dynamic website system where content and product details could be fully managed through an admin dashboard, storing and retrieving data from a MySQL database. This taught me how to balance frontend flexibility with backend control, while reinforcing my understanding of CRUD operations, data validation, and content management workflows.

Through this project, I gained hands-on experience with:

  • Structuring large datasets efficiently
  • Building scalable admin interfaces
  • Writing clean, maintainable backend logic
  • Applying real-world web development practices
